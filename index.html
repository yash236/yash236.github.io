Dot Net FrameWork

If any programming language adheres to the standard followed by .NET, it can be considered part of the .NET ecosystem.

Certainly! The .NET ecosystem is built upon a set of standards and specifications known as the Common Language Infrastructure (CLI). The CLI defines a common execution environment for programming languages and provides a runtime environment for executing programs.

When a programming language follows the CLI standard, it gains compatibility with the .NET framework and can be used to develop applications that target the .NET platform. This means that developers can write code in different languages while still leveraging the capabilities and libraries provided by .NET.

Some popular languages that conform to the CLI standard and can be used within the .NET ecosystem include C#, Visual Basic .NET (VB.NET), and F#. Additionally, there are managed versions of other languages like C++ and Python that can also be used with .NET.

By adhering to the CLI standard, these languages ensure interoperability and enable code sharing between different .NET languages. This allows developers to leverage the strengths of each language while building applications for the .NET platform.

In summary, any programming language that adheres to the CLI standard can be considered part of the .NET ecosystem, enabling developers to build applications that target the .NET framework using their language of choice.

CLR

CIL/MSIL file gnerated from compiler CLR(common language runtime) after compile JIT is part of CLR only execute main code(Fast Compile)

The Common Language Runtime (CLR) is a component of the .NET framework that provides a managed execution environment for applications. When a program is compiled, the compiler generates an intermediate language representation called Common Intermediate Language (CIL) or Microsoft Intermediate Language (MSIL).

After compilation, the CIL/MSIL code is not directly executed by the hardware or operating system. Instead, it is executed by the CLR. The CLR is responsible for loading and executing the CIL/MSIL code. It provides services such as memory management, exception handling, security, and garbage collection.

When the CIL/MSIL code is executed, the Just-In-Time (JIT) compiler of the CLR dynamically compiles the CIL/MSIL code into machine code specific to the target platform. This compilation happens at runtime, just before the code is executed. The JIT compiler optimizes the code for better performance and generates native machine code that can be executed directly by the hardware.

The JIT compilation process allows for faster startup times and efficient execution of the code. It compiles only the portions of the code that are actually executed, which is known as "just-in-time" compilation.

To summarize, the CLR executes the CIL/MSIL code generated by the compiler. The JIT compiler within the CLR dynamically compiles the CIL/MSIL into machine code at runtime, optimizing it for performance. This combination of the CLR and JIT compilation enables the execution of managed code in a platform-independent manner while providing performance benefits.

Namespace

A namespace is used to organize code in a structured manner. It contains classes, interfaces, enums, delegates, and structs.

A namespace in programming is a way to organize code elements such as classes, interfaces, enums, delegates, and structs into logical groups. It provides a hierarchical structure to prevent naming conflicts and helps in maintaining code readability and organization.

By placing related code elements within a namespace, you can group them based on their functionality, purpose, or module. This allows you to have multiple code elements with the same name, as long as they are in different namespaces.

Namespaces also provide a level of encapsulation, as code elements within a namespace are not directly accessible outside of it unless explicitly specified. This helps in managing the visibility and accessibility of code elements within a project or application.

Using namespaces, you can reference and access code elements by specifying the fully qualified name, which includes both the namespace and the code element name. This ensures clarity and avoids ambiguity when referring to specific code element

Reading & Writing in Console

In C#, you can read and write input/output to the console using the Console class.

To write output to the console, you can use the Console.Write() or Console.WriteLine() methods. Console.Write() writes the specified value to the console without a newline character at the end, while Console.WriteLine() writes the specified value to the console followed by a newline character. 

    // Writing output to the console
    Console.Write("Hello ");
    Console.WriteLine("World");

To read input from the console, you can use the Console.ReadLine() method. This method reads the next line of characters from the console and returns it as a string. 

    // Reading input from the console
    Console.Write("Enter your name: ");
    string name = Console.ReadLine();
    Console.WriteLine("Hello, " + name);

Float, Decimal, Double

Float takes 4 bytes space and it can take up 8 character of limit in it including dot. We need to use 'f' as suffix (Mandatory)

Double takes 8 bytes space and it can take up 16 character of limit in it including dot. We need to use 'd' as suffix(Optional)

Decimal takes 16 bytes space and it can take up 32 character of limit in it including dot. We need to use 'm' as suffix(Mandatory)
